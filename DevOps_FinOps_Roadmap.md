# 🚀 CoverChallenge 데브옵스/FinOps 마스터 플랜 및 컨텍스트 인수인계

> **[Agent Context Handoff]**
> 이 파일은 새로운 채팅 세션에서 AI 에이전트(FinOps 튜터)가 이전 대화의 맥락과 앞으로의 로드맵을 즉시 파악하기 위해 작성된 인수인계 문서입니다.
> - **현재 상태:** 로컬 환경에서 `docker-compose.yml`을 통해 Spring Boot, MySQL, Nginx가 내부 네트워크로 격리되어 실행되는 1단계 도커라이징이 완료됨.
> - **기존 AWS 구축 내역:** 이전 대화에서 이미 AWS 자원(EC2, RDS, S3 등)을 일부 구축하고 다뤄본 실습 히스토리가 있음.
> - **AWS 계정 상태 (FinOps 이슈):** 단, 현재 일반 프리티어 계정이 만료된 상태이므로, 비용 폭탄을 막기 위해 철저한 FinOps 관점의 접근이 필요함. 학교 계정(AWS Educate/Academy)을 통해 크레딧을 지원받아 기존 자원을 이관하거나 새롭게 최적화하여 진행할 예정임.
> - **최종 목표:** 겉핥기식 기술 사용이 아닌, OS/네트워크 기초부터 클라우드/FinOps/CI-CD 아키텍처까지 완벽히 이해하는 '시니어 데브옵스 엔지니어' 수준의 인프라 구축.

---

## 🗺️ 근본부터 다지는 데브옵스 4단계 로드맵

### [Phase 1] 운영체제(Linux)와 네트워크의 본질 (현재 단계)
- **왜 하는가?** 클라우드나 도커를 쓰기 전에, 밑바탕이 되는 리눅스 커널과 TCP/IP 네트워크의 원리를 모르면 장애(Troubleshooting) 발생 시 손도 댈 수 없습니다.
- **실습 목표:**
  1. 현재 작성된 `docker-compose.yml`의 `internal` 브릿지 네트워크가 내부적으로 어떻게 IP를 할당하는지(DNS 해상도) 원리 파악.
  2. Nginx의 리버스 프록시(Reverse Proxy)가 80포트를 8080포트로 전달하는 패킷 흐름의 이해.
  3. 보안: DB(3306)와 Spring Boot(8080) 포트를 외부에서 막아야 하는 이유 체득.

### [Phase 2] 컨테이너 격리 기술의 딥다이브 (Docker)
- **왜 하는가?** '그냥 남들이 쓰니까' 도커를 쓰는 것이 아니라, 가상머신(VM)과의 차이점을 명확히 알고 커널 자원 격리를 이해하기 위함입니다.
- **실습 목표:**
  1. 리눅스 `cgroup`과 `namespace` 관점에서의 도커 이해.
  2. 현재의 `Dockerfile`을 해부하여, 레이어(Layer) 캐싱을 통한 빌드 속도 단축 및 멀티 스테이지 빌드(Multi-stage build)로 이미지 용량 최소화하기.
  3. 도커 볼륨(Volume)을 통한 DB 데이터 영속성 보장 실습.

### [Phase 3] 클라우드 인프라와 FinOps 최적화 (AWS)
- **왜 하는가?** 단순히 EC2에 올리는 것을 넘어, 비용(Cost)을 최소화하면서도 안정적인 인프라를 구축하는 아키텍트의 시야를 기릅니다.
- **실습 목표:**
  1. AWS VPC, 퍼블릭/프라이빗 서브넷, 보안그룹(SG)을 통한 네트워크 격리망 직접 설계.
  2. **[FinOps 핵심]** 1GB 램을 가진 EC2 프리티어에서 3개의 컨테이너가 돌다 터지는 OOM(Out of Memory) 현상 경험하기.
  3. 서버 스펙을 돈 주고 올리지 않고, 스왑(Swap) 메모리를 잡아 해결하는 데브옵스 기술 실습.
  4. AWS CloudWatch 예산 알림(Budget) 설정으로 요금 폭탄 원천 차단.

### [Phase 4] 무중단 배포 및 자동화 (CI/CD)
- **왜 하는가?** 개발자가 코드를 수정할 때마다 서버에 접속해 명령어를 치는 반복 노동(Toil)을 없애고, 에러 배포를 차단합니다.
- **실습 목표:**
  1. GitHub Actions를 연동해 소스코드 Push 시 자동 빌드 및 테스트(CI) 구축.
  2. 빌드된 이미지를 Docker Hub에 자동 Push 후 EC2 서버에서 Pull 받는 배포(CD) 파이프라인.
  3. 배포 중 서버가 다운되지 않도록 무중단 배포(Blue/Green 혹은 Rolling) 스크립트 작성.
